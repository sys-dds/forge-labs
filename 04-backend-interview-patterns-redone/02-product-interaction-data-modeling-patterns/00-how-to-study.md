# How To Study Product Interaction Modeling

Start with the interview prompt, read the data model, then run the proof. After the proof passes, inspect the trap rows: blocked viewers, inactive users, removed follows, pending friend requests, inactive reactions, deleted comments, reports, unmatched pairs, muted feed authors, self-notification events, private activity rows, already-followed recommendation candidates, stale counters, close-friends membership rows, duplicate bookmarks, viewer-specific hides, evidence rows, policy decisions, treatments, appeals, reversals, trust components, abuse signal evidence, creator exposure budgets, buyer intent rows, seller quality components, offer/booking lifecycle rows, supply/demand snapshots, conversation participants, read receipts, message deletion rows, inbox settings, community memberships, role permissions, join requests, invites, channel memberships, post moderation actions, event RSVPs, waitlists, attendance rows, hashtags, search filters, saved searches, trend components, collection items, and capstone rows that combine several rules.

For each clinic, say the write path out loud before the read query. Interviewers listen for constraints and lifecycle states as much as SQL syntax. For clinics 006-010, also say why a visible row appears and why each hidden row is excluded. For clinics 021-025, explain the row-level reason: exposure budget, buyer intent, seller component, booking lifecycle, supply/demand, or marketplace treatment. For clinics 026-030, explain the per-user reason: participant state, read marker, deletion scope, mute/archive setting, block, report, or inbox tie-breaker. For clinics 031-035, explain the community-specific reason: active membership, ban, role permission, request or invite state, channel membership, post moderation state, RSVP state, waitlist state, attendance, or audit trace. For clinics 036-040, explain the discovery-specific reason: normalized topic, followed topic, search filter, facet count, saved-search state, trend component, safety treatment, collection visibility, item lifecycle, or debug trace.
