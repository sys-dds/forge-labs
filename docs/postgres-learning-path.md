# PostgreSQL Learning Path

`02-postgres` comes after `01-sql` because the learner already knows how to shape rows. This path asks which rules PostgreSQL should own: exact types, constraints, upserts, JSONB shape, read models, functions, triggers, tenant policies, worker coordination, and namespace hygiene.

Recommended order: run chapters first, then practice packs, then gauntlets, then interview drills, then self-assessment. Move to database optimisation only after you can prove the correct rows before discussing speed.
