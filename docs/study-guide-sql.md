# SQL Study Guide

Study one chapter at a time. Do not skim all folders horizontally.

1. Read the chapter README and explainer.
2. Read every solution file in numeric order.
3. Run the chapter proof.
4. Read the common mistakes.
5. Break one constraint or query deliberately.
6. Fix it.
7. Add one small query from the extension tasks.

The goal is to connect syntax to backend behavior: discovery, correctness, visibility, counting, pagination, candidate generation, and matching.
