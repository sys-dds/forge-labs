# SQL to PostgreSQL Transition

Move to PostgreSQL-specific study after the SQL mastery verification passes and your misses are no longer about SQL meaning.

Ready signals:

- you can repair gauntlets without reading the solution first
- take-home proofs pass locally
- interview answers name exact rows and trade-offs
- remaining questions are about indexes, query plans, locking, data types, constraints, or PostgreSQL-specific features

Stay in SQL if join direction, aggregation grain, cursor ordering, recursive scope, or candidate exclusions still feel shaky.
