# SQL Skill Rubric

1. Can read the query: identify tables, joins, filters, and output columns.
2. Can run and inspect the data: load a pack and find named rows in Adminer, DbGate, or psql.
3. Can explain why each row appears: name the join path for positive rows.
4. Can identify why rows are excluded: name the predicate or constraint that removes each negative row.
5. Can fix a broken query: repair a gauntlet without changing the proof.
6. Can write proof tests: assert exact rows, counts, ordering, and exclusions.
7. Can explain trade-offs in interview language: discuss grain, safety, ordering, and constraints clearly.
8. Ready to move to PostgreSQL: SQL proofs pass locally and remaining misses are about engine behavior, indexing, plans, or concurrency rather than query meaning.
