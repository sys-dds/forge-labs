# PostgreSQL Learning Path

1. Model backend facts with PostgreSQL-specific types.
2. Move durable correctness into constraints, partial uniqueness, and exclusion rules.
3. Use upserts for retry-safe writes.
4. Use JSONB for documented flexible payloads while keeping core facts typed.
5. Shape reads with views and materialized views.
6. Encapsulate narrow rules with functions and controlled mutations.
7. Use triggers only when audit or timestamp behavior truly belongs beside the row.
8. Prove tenant boundaries with RLS.
9. Coordinate local workers with advisory locks and lightweight notifications.
10. Keep schemas and search paths explicit before moving to optimisation.
